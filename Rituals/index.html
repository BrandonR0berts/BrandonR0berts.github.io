Main Concepts Developed

Pin System.

As the project was planned to have a fair amount of levels there, therefore, needed to be a way to efficiently traverse the multiple worlds and levels in an intuitive manner. My solution to this was to develop an abstract 'Pin' system for level and world navigation.

This system allowed for a texture to be selected, which defined the collision for the selection, along with what type of level this was, be it move capped, time restricted, or otherwise. This system also allowed for modification of the amount of enemies within the level, along with the starting health they have.

Furthermore, when the level is selected the settings stored in the 'Pin' are displayed and passed into the loading level when the level is entered. This system also accomidates setting that the 'pin' loads into another world map, instead of a level. This allows for the world map screen traversion.

Tutorials.

During development I was responsible for creating and implementing the majority of the tutorial levels, including the board piece layout within the tutorial levels, and the explanation sequencing. This required ensuring that there were no accidentally possible moves that could ruin the intended way of playing through the level. As this would have locked the player in the level without a way to exit.
Creating the tutorials mostly consisted of designing the initial idea and layout, and they playtesting repeatly to ensure there were no unintential movements possible.

Audio Systems.

There is a lot of Music and SFX within rituals, ranging from the tiered audio tracks within boss fights, to the button press sounds. This audio all required individual triggers within Unity's audio system, therefore developing a fade in/out system that has all audio slowly reduce or increase in volume is a sizable task. In order to handle this I used an 'Audio Manager' object that was persists accross all scenes and controls all audio triggering. Having all audio go through a system like this allows for central control of the volume of all audio clips simultaniously. Allowing for the muting of all audio in the settings, along with the fade in/out when transitioning between levels. Adding SFX, audio, and fade in/out audio effects

Quality Assurance.

Towards the end of development there were very few aspects of the game that still required coding, so my job shifted to working with the QA tester to hunt down any missed bugs. And, if any were found, fixing them using the re-creation steps provided.
